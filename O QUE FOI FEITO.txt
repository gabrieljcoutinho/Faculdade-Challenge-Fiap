Realizei um projeto acadêmico com base em uma proposta apresentada pela empresa GoodWe, do setor de energia solar, durante uma visita à faculdade.
O objetivo da atividade era identificar os principais desafios enfrentados pela empresa e desenvolver soluções tecnológicas que pudessem melhorar seus processos.
As propostas podiam variar entre sistemas completos ou protótipos simples, utilizando tecnologias como React, Python ou até inteligência artificial.

Diante disso, desenvolvi uma aplicação completa com React, simulando um aplicativo mobile para gerenciamento e automação de recursos relacionados à energia solar.
 O projeto foi publicado na web, permitindo o acesso público por meio de um link.

Estrutura do Sistema
A aplicação conta com um cabeçalho fixo que direciona o usuário para as seguintes seções: Home, Conexões de Aparelhos, Chatbot, Contato e Configurações.

Home
Nesta seção, o usuário pode alternar entre três tipos de gráficos (linha, barras e pizza) onde há um botao que mostra a produção do gráfico
sem ser pelo chat ta,bem, gerados com base em dados de uma API simulada.
Abaixo dos gráficos, são exibidas informações do clima atual e previsão do tempo. Ao final da página, há uma área dedicada a destacar os
 impactos positivos do uso de energia solar, com foco nos benefícios econômicos e ambientais.. Atraz do gráfico, há uma imgagem simbolizando
o período do dia, se for de manhã vai ser um sol mais fraco, se for de tarde um sol masi forte e e for À noite nã ovai ter sol

Conexões de Aparelhos
O usuário pode adicionar dispositivos de duas formas: via conexão Bluetooth (simulada) ou manualmente. Na adição manual,
é possível escolher o nome, ícone e cor de fundo do dispositivo. Após o cadastro, o aparelho aparece como conectado,
 com opções de gerar QR Code (para facilitar o compartilhamento), editar, excluir ou desativar.
Os dispositivos conectados são organizados em duas listas: conectados e desconectados. Quando um aparelho é desativado,
 ele é automaticamente movido para a seção de desconectados, podendo ser reativado a qualquer momento. Também é possível reordenar os dispositivos arrastando-os na tela.
 Na tela há uma imagem que msotra que nã otem nenhum aprelho coenctado para o usuário sber e para não deixar simplesmente em branco.

Chatbot
Foi implementado um sistema de mensagens com recursos básicos de chat. Além disso, o chatbot integra uma API
 personalizada de perguntas e respostas específicas, e uma API de inteligência artificial para respostas gerais. Ele reconhece
 comandos como “conectar aparelho” ou “mudar de página”, interagindo como um assistente virtual.

Contato
Inclui um formulário para envio de mensagens, com campos para nome, e-mail e conteúdo. Também há a opção de enviar arquivos
 e acessar uma seção de perguntas frequentes, facilitando a comunicação com a empresa.


Configurações
A seção reúne seis funcionalidades principais:
Lista de todos os comandos reconhecidos pelo sistema;
Central de ajuda;
Leitura em voz alta de textos clicados, promovendo acessibilidade;
Alternância entre modo claro e escuro;
Login e logout com autenticação fake via e-mail/senha, Google ou Facebook,
Página de recuperação de senha (em desenvolvimento).
Considerações Finais
O sistema foi desenvolvido para funcionar tanto em dispositivos móveis quanto em desktops, com foco no uso mobile para facilitar o acesso remoto.
As APIs utilizadas são fictícias, pois não há acesso direto aos dados reais da empresa.
O projeto também conta com animações suaves, especialmente visíveis nas transições de página, como um efeito de "apagando e acendendo" ao navegar entre seções,
 contribuindo para uma experiência mais fluida e agradável. O código-fonte está disponível publicamente para consulta.
